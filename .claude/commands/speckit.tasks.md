# Spec-Kit: Tasks

Generate a phased task list from the current branch's plan and specification.

## Pre-Flight

### Current branch
!`$HOME/.claude/hooks/speckit-helper.sh branch`

### Auto-detect spec branch
!`$HOME/.claude/hooks/speckit-helper.sh check-plan`

### Load spec
!`$HOME/.claude/hooks/speckit-helper.sh spec`

### Load plan
!`$HOME/.claude/hooks/speckit-helper.sh plan`

## Instructions

**Required inputs**: Both `plan.md` and `spec.md` must exist for the current branch. If either is missing, tell the user which command to run first.

1. **Load artifacts**:
   - Read `spec.md` for user scenarios (US#) and functional requirements (FR-NNN)
   - Read `plan.md` for affected files, implementation approach, and design decisions

2. **Generate tasks in phases**:

   Write `.specify/specs/<branch>/tasks.md`:

   ```markdown
   # Tasks: <branch-name>
   <!-- Generated by /speckit.tasks -->
   <!-- Date: YYYY-MM-DD -->

   ## Phase 1: Setup
   - [ ] T001 [US1] <setup task description> `path/to/file`
   - [ ] T002 [US1] <setup task description> `path/to/file`

   ## Phase 2: Foundational
   - [ ] T003 [FR-001] <data model/schema task> `path/to/file`
   - [ ] T004 [P] [FR-002] <foundation task> `path/to/file`

   ## Phase 3: User Stories (by priority)
   ### P1 — Must Have
   - [ ] T005 [P] [US1] <implementation task> `path/to/file`
   - [ ] T006 [US1] <implementation task> `path/to/file`

   ### P2 — Should Have
   - [ ] T007 [US2] <implementation task> `path/to/file`

   ### P3 — Nice to Have
   - [ ] T008 [US3] <implementation task> `path/to/file`

   ## Phase 4: Polish
   - [ ] T009 [US1] <integration test> `path/to/file`
   - [ ] T010 <documentation/cleanup> `path/to/file`

   ---
   **Legend**: `[P]` = parallelizable | `[US#]` = user scenario | `[FR-NNN]` = functional requirement
   ```

3. **Task format rules**:
   - ID format: `T001`, `T002`, etc. (zero-padded 3 digits)
   - `[P]` marks tasks that can run in parallel with adjacent tasks
   - Every task references at least one US# or FR-NNN
   - Every task includes a target file path
   - Tasks within a phase are ordered by dependency

4. **Create TaskCreate entries** in the Claude Code task tracker:
   - One TaskCreate per task from the generated tasks.md
   - Subject: `[T001] <description>`
   - Description: include the user scenario reference, file path, and acceptance criteria from the spec
   - Set dependencies with TaskUpdate `addBlockedBy`/`addBlocks` where tasks have ordering requirements
   - Phase 1 tasks have no dependencies
   - Phase 2 tasks are blocked by Phase 1
   - Phase 3 tasks are blocked by Phase 2
   - Phase 4 tasks are blocked by Phase 3

5. **Present summary** to user:
   - Total task count by phase
   - Parallelizable task count
   - Estimated phases (sequential dependency chain length)
   - Suggest next step: `/speckit.implement` or `/speckit.checklist` for pre-implementation validation
